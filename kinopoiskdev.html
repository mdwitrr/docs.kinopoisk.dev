kinopoisk.dev

Документация к API кинопоиска kinopoisk.dev

API позволяет получать максимум информации с кинопоиска о фильмах, сериалах и персонах. Всего в api 5 моделей. Искать можно по любому полю в эти моделях. Ниже приведены примеры использования api. Это лишь примеры, а не все возможности api. Не бойтесь эксперементировать.

Тестовый токен в этой документации имеет ограничение на 200 запросов в день.

Для получения личного токена и при возникновении проблем пишите в телеграме https://t.me/mdwit

А технические вопросы по языкам программирвоания и api можете задавать в нашем чате для разработчиков https://t.me/dev_to_dev

GET /movie ¶

Movie хранит в себе фильмы, сериалы, мультфильмы, мультсериалы и аниме. То есть тут все картины с кинопоиска. Подробная схема модели содержит информацию о каждом ключе.

Доступные query параметры модели movie:

  • Parameters
    • token: (string) - Токен для авторизации
    • search: (string) - Искомое значение. Может быть как словом, частью слова, буквой, целым числом, так диапозоном значений. Так же важно понимать понимать, что диапазоны будут работать только для полей типа Number и Date. Диапозон может содаржать 2 значения и разделять их нужно через символ -. Чтобы найти фильмы с рейтингом от 7 до 10 нужно указать 7-10. Чтобы найти фильмы с датой релиза от 01.02.2020 и до 03.02.2021 нужно указать 01.02.2020-03.02.2021 и можно указать просто года премьеры 2020-2021
    • field: (string) - Ключ по которому необходимо искать. Полей может быть несколько, но на каждый ключ должен быть дополнительный search. Парметры field образовывают один объект с параметрами search
    • page: (string) - Номер страницы пагинации default - 1
    • limit: (string) - Количество результатов на страницу default - 10
    • sortField: (string) - Ключ по которому необходимо сортировать. Логика работы такая же как как и атрибутом field для поиска
    • sortType: (string) - Поддерживает 2 значения 1 и -1. 1 - сортировка по возрастанию, а -1 в порядке убывания

Поиск  ¶

Используя поиск вы можете найти совершенно любой фильм по любому полю схемы. Поиск очень гибкий и может показаться сложным 😀

Внимание: параметр field должен содержать ключ по которому необходимо произвести поиск. Ключ должен сохранять вложенность.
Например: если вы хотите искать по id фильма, в параметр field передаем только id field=id, так как ключе id он находится в корне объекта.
Но если нужно найти рейтинг кинопоиска, то необходимо указывать что мы ищем в rating значение ключа kp и разделять это все точкой field=rating.id.

Ниже я постараюсь привести примеры запрсово исходя из ответов в опроснике.
